================================================================================
 üêõ TROUBLESHOOTING
================================================================================

ISSUE: conda command not found
FIX: 
  # Mac/Linux
  export PATH="$HOME/miniconda3/bin:$PATH"
  source ~/.bashrc
  
  # Windows
  Use Anaconda Prompt

ISSUE: Environment creation is slow
FIX:
  # Install mamba (faster)
  conda install mamba -n base -c conda-forge
  mamba env create -f environment.yml

ISSUE: GPU not detected
FIX:
  # Check CUDA version
  nvidia-smi
  
  # Reinstall PyTorch with correct CUDA
  conda activate yolov11-wildlife
  conda install pytorch torchvision pytorch-cuda=11.8 -c pytorch -c nvidia

ISSUE: PackagesNotFoundError
FIX:
  conda config --add channels conda-forge
  conda config --add channels pytorch

================================================================================
 ‚úÖ CONDA VS PIP COMPARISON
================================================================================

USE CONDA IF:
  ‚úì You need GPU support (easier CUDA setup)
  ‚úì You use scientific packages (NumPy, SciPy, etc.)
  ‚úì You want system-level dependency management
  ‚úì You prefer comprehensive environment isolation

USE PIP (venv) IF:
  ‚úì You prefer lightweight setup
  ‚úì You already have CUDA configured manually
  ‚úì You want faster environment creation
  ‚úì You're on a system with limited resources

BOTH ARE SUPPORTED:
  - Conda: environment.yml + setup_conda_and_train.sh
  - Pip:   requirements.txt + setup_and_train.sh
